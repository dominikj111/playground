🚀 Welcome to Django REST API

This project demonstrates Django REST Framework best practices with JWT authentication.

Your background: PHP, ColdFusion, TypeScript/Express - this will feel familiar!

🔐 Authentication Endpoints

Get your JWT token to access protected endpoints:

POST /api/token/ Login - Get access & refresh tokens
POST /api/token/refresh/ Refresh access token
POST /api/register/ Register new user (public)

👥 User Management (ViewSet)

RESTful CRUD operations using ViewSets (like Laravel Resource Controllers):

GET /api/users/ List all users (requires auth)
GET /api/users/{id}/ Get specific user
GET /api/users/me/ Get current user (custom action)
POST /api/users/ Create user
PUT /api/users/{id}/ Update user
DELETE /api/users/{id}/ Delete user

🔧 Utility Endpoints

GET /api/public/ Public endpoint (no auth required)
GET /api/protected/ Protected endpoint (requires JWT)
GET /api/health/ Health check
POST /api/echo/ Echo back your request

📝 Quick Start

1. Register a user:

POST /api/register/ with {"username": "user", "email": "user@example.com", "password": "securepass123", "password_confirm": "securepass123"}

2. Get JWT token:

POST /api/token/ with {"username": "user", "password": "securepass123"}

3. Use the token:

Authorization: Bearer {your_access_token}
